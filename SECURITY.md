# Security Policy

## Supported Versions

The following versions of **MorphNavBar** are currently supported with security fixes, stability improvements, and important maintenance updates.

| Version | Supported |
| ------- | --------- |
| 2.2.x   | :white_check_mark: |
| 2.1.x   | :white_check_mark: |
| 2.0.x   | :white_check_mark: |
| < 2.0   | :x: |

Older versions may continue to work, but they are no longer actively maintained or guaranteed to receive fixes.

---

## Reporting a Vulnerability

If you discover a security issue, vulnerability, or behavior that may negatively affect applications using **MorphNavBar**, please report it responsibly.

Please include as much detail as possible:

- A clear description of the issue
- Steps to reproduce the problem
- Affected version(s)
- Android version / device details (if relevant)
- Proof of concept or sample project (if available)
- Possible impact of the issue

## How to Report

Please report vulnerabilities privately by contacting me through:

- GitHub Issues *(for non-sensitive reports only)*
- GitHub profile contact methods
- Direct message if available

For sensitive issues, please avoid posting public details before the problem is reviewed and fixed.

---

## Response Process

After a report is received:

- Initial review will usually happen as soon as possible
- You may be contacted for additional details
- If confirmed, a fix will be prepared for a supported version
- A release update may be published when ready

---

## Scope

MorphNavBar is a UI library, so typical security concerns may include:

- Crashes caused by malformed input
- Unsafe public API behavior
- Resource abuse or denial-of-service via repeated misuse
- Accessibility or unexpected interaction issues
- Dependency-related risks (if introduced in future versions)

---

## Responsible Disclosure

Please allow reasonable time for investigation and fixes before publicly disclosing vulnerabilities.

Thank you for helping improve the security and reliability of MorphNavBar.
