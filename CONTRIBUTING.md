# Contributing to MorphNavBar

Thank you for your interest in contributing to MorphNavBar.

MorphNavBar is a custom Android navigation bar built with a fully manual Canvas-based rendering approach, so contributions should keep the animation, drawing behavior, XML attributes, and public API stable.

## How You Can Contribute

You can help by:

- Reporting bugs
- Suggesting feature improvements
- Fixing issues in the source code
- Improving documentation and examples
- Improving animation, accessibility, or performance
- Helping keep the library stable and easy to use

## Before You Start

Before opening an issue or pull request:

- Search existing issues first
- Make sure the change fits the current design of the library
- Read the README and usage examples carefully
- Keep your change focused and easy to review

## Reporting Bugs

Please include:

- A clear description of the problem
- Steps to reproduce it
- Expected behavior
- Actual behavior
- Device and Android version
- Library version
- Relevant code snippets or screenshots if possible

## Requesting Features

For feature requests, please describe:

- What you want to add or improve
- Why it is useful
- How it should behave visually and functionally
- Any animation or UI details that matter

## Pull Requests

Pull requests are welcome.

Please keep them:

- Small and focused
- Consistent with the existing code style
- Compatible with the current XML attributes and public methods
- Safe for existing projects that already use the library

## Code Style

Please follow the existing project style:

- Use clear and readable Java code
- Keep the view drawing logic maintainable
- Avoid breaking existing attributes or behavior
- Preserve smooth animation and visual polish
- Keep changes compatible with the current API

## Documentation

If you change:

- XML attributes
- Public methods
- Default values
- Behavior of animations or callbacks

please update the README and related examples as well.

## Need Help?

If the change is large or affects the public API, please open an issue first.
